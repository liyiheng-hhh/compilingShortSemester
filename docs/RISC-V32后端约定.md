# RISC-V32 后端约定文档

## 1. 寄存器使用约定

### 1.1 寄存器分类

| 寄存器 | 名称 | 使用约定 | 调用者保存 |
|--------|------|----------|-----------|
| x0 | zero | 恒为 0，不可修改 | - |
| x1 | ra | 返回地址 | 被调用者保存 |
| x2 | sp | 栈指针 | 被调用者保存 |
| x3 | gp | 全局指针 | - |
| x4 | tp | 线程指针 | - |
| x5-x7 | t0-t2 | 临时寄存器 | **调用者保存** |
| x8 | s0/fp | 帧指针 | 被调用者保存 |
| x9 | s1 | 保存寄存器 | 被调用者保存 |
| x10-x17 | a0-a7 | 参数/返回值寄存器 | **调用者保存** |
| x18-x27 | s2-s11 | 保存寄存器 | 被调用者保存 |
| x28-x31 | t3-t6 | 临时寄存器 | **调用者保存** |

### 1.2 函数调用约定

- **参数传递**：前 8 个参数依次放入 `a0-a7`，超过 8 个的参数通过栈传递
- **返回值**：`int` 返回值放入 `a0`，`void` 函数不使用返回值
- **调用者责任**：调用函数前需保存 `t0-t6`、`a0-a7`（如果后续还要使用）
- **被调用者责任**：进入函数后需保存 `ra`、`s0-s11`（如果使用了）

### 1.3 汇编生成时的寄存器分配

- **表达式计算**：优先使用 `t0-t2` 作为临时寄存器
- **变量存储**：局部变量使用栈，全局变量使用 `.data` 段
- **函数参数**：直接从 `a0-a7` 读取，必要时存入栈

## 2. 栈帧布局

### 2.1 标准栈帧结构

```
高地址
┌─────────────────────────────────────┐
│         参数溢出区（第9+个参数）       │
├─────────────────────────────────────┤
│      返回地址 ra（x1）                │  ← sp + 4 * (n + 1)
├─────────────────────────────────────┤
│      帧指针 s0/fp（x8）              │  ← sp + 4 * n
├─────────────────────────────────────┤
│      保存寄存器 s1-s11（如使用）       │
├─────────────────────────────────────┤
│      局部变量区                       │
├─────────────────────────────────────┤
│      临时空间/栈参数传递区             │
├─────────────────────────────────────┤
│      被调用函数参数（前8个用a0-a7）     │
低地址  ← sp（栈指针）
```

### 2.2 栈帧偏移计算

假设函数有 `k` 个局部变量，使用了 `m` 个保存寄存器：

- 栈大小 = 4 * (2 + m + k) （2 = ra + fp）
- 局部变量 `i` 的偏移 = 4 * (2 + m + i) （相对于 fp）
- 返回地址偏移 = -4 （相对于 fp）
- 被调用者参数在栈上的偏移 = 0, 4, 8, ... （相对于 sp）

### 2.3 函数入口/出口模板

```asm
函数名:
    # 函数入口
    addi    sp, sp, -栈大小
    sw      ra, 0(sp)
    sw      s0, 4(sp)
    addi    s0, sp, 栈大小     # 设置帧指针
    
    # ... 函数体 ...
    
    # 函数出口
    lw      s0, 4(sp)
    lw      ra, 0(sp)
    addi    sp, sp, 栈大小
    ret
```

## 3. 汇编输出格式

### 3.1 段结构

```asm
    .data
    # 全局变量初始化
    .globl  global_var
global_var:
    .word   100
    
    .text
    .globl  main
main:
    # 代码...
    ret
```

### 3.2 全局数据区（.data）

- 全局变量：`.word` 初始化值，`.zero` 未初始化
- 全局常量：与全局变量相同，但语义上不可修改

### 3.3 代码区（.text）

- 每个函数以 `.globl 函数名` 声明全局可见
- 函数名作为标签，后跟冒号
- 指令前有 4 空格缩进

### 3.4 注释

- 使用 `#` 开头的单行注释
- 函数入口处注明函数名和参数
- 关键控制流跳转处注明目标

## 4. 标签命名规范

### 4.1 函数标签

- 使用函数的原始名称
- 示例：`main`, `add`, `factorial`

### 4.2 全局变量标签

- 使用变量的原始名称
- 示例：`global_var`, `CONSTANT`

### 4.3 局部标签

- 格式：`.L数字_描述`
- 示例：`.L0_if_true`, `.L1_loop_begin`, `.L2_while_cond`

### 4.4 临时标签

- 格式：`.Ltemp_数字`
- 示例：`.Ltemp_0`, `.Ltemp_1`

## 5. IR 到汇编映射规则

### 5.1 表达式映射

| IR 节点 | 汇编指令 | 说明 |
|---------|----------|------|
| 常量 | `li rd, 常量值` | 加载立即数 |
| 标识符引用 | `lw rd, 偏移(fp)` | 从栈加载局部变量 |
| 全局变量引用 | `lw rd, 0(global_var)` | 从全局数据区加载 |
| 一元运算 | `neg`, `not` 等 | 根据操作符生成对应指令 |
| 算术运算 | `add`, `sub`, `mul`, `div`, `rem` | 生成对应算术指令 |
| 关系运算 | `slt`, `sgt`, `sle`, `sge`, `seq`, `sne` | 生成比较指令，结果为 0 或 1 |
| 逻辑运算 | `beqz` + 跳转 | 短路求值需配合分支 |

### 5.2 控制流映射

| IR 节点 | 汇编指令 | 说明 |
|---------|----------|------|
| if | `beqz` / `bnez` + 标签跳转 | 条件判断后跳转 |
| while | 标签 + `beqz` / `bnez` + 跳转 | 循环条件检查 + 循环体 |
| break | `j` 跳转到循环结束标签 | 无条件跳转 |
| continue | `j` 跳转到循环条件标签 | 无条件跳转 |
| return | 计算返回值 → `j ra` | 将结果放入 a0，返回 |

### 5.3 函数调用映射

```asm
    # 参数准备
    li      a0, 参数1
    li      a1, 参数2
    # ...
    
    # 保存调用者保存的寄存器（如果需要）
    sw      t0, 0(sp)
    
    # 调用
    call    函数名
    
    # 恢复调用者保存的寄存器
    lw      t0, 0(sp)
    
    # 返回值在 a0 中
```

## 7. 调试方法

### 7.1 汇编检查

- 使用 `riscv32-unknown-elf-as` 汇编器检查语法
- 使用 `riscv32-unknown-elf-objdump` 反汇编查看生成结果

### 7.2 模拟运行

- 使用 Spike 或 QEMU 运行生成的汇编
- 检查 `a0` 寄存器的值是否符合预期

### 7.3 常见问题定位

| 现象 | 可能原因 | 排查方法 |
|------|----------|----------|
| 返回值错误 | 表达式计算错误 | 检查每条算术指令的操作数和结果寄存器 |
| 程序崩溃 | 栈溢出或寄存器未保存 | 检查栈帧大小和保存/恢复指令 |
| 跳转错误 | 标签名错误或条件判断错误 | 检查 `beqz`/`bnez` 的条件和目标标签 |
| 变量值错误 | 偏移计算错误 | 检查 `lw`/`sw` 的偏移量是否正确 |

## 8. 协作接口

### 8.1 与前端（李）的接口

- 输入：表达式 AST 节点
- 输出：对应表达式的汇编代码片段
- 约定：表达式结果放在 `t0` 寄存器中

### 8.2 与语义分析（张）的接口

- 输入：符号表（变量类型、作用域、存储位置）
- 输出：变量访问的汇编代码（栈偏移或全局标签）
- 约定：局部变量使用栈偏移，全局变量使用 `.data` 段标签

### 8.3 与控制流（贾）的接口

- 输入：控制流 AST 节点（if/else/while/break/continue/return）
- 输出：对应控制流的汇编代码（分支指令、标签、跳转）
- 约定：标签命名遵循第 4 节规范，跳转指令使用 `beqz`/`bnez`/`j`
