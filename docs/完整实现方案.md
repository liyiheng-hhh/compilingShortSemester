# ToyC 编译器完整实现方案

## 1. 总体路线

项目目标是实现一个从 ToyC 源码到 RISC-V32 汇编的编译器。编译器从 `stdin` 读取源码，从 `stdout` 输出汇编，并接受可选参数 `-opt` 开启优化。

本分支采用以下技术路线：

```text
stdin
  -> Lexer
  -> Parser
  -> AST
  -> scoped symbol/const handling
  -> optional constant folding
  -> RISC-V32 Code Generator
  -> stdout
```

实现语言使用 C++20，构建系统使用 CMake。语法分析采用手写递归下降，不依赖 ANTLR/Flex/Bison。原因是 ToyC 文法规模适中，手写 Parser 更容易控制错误位置、AST 结构和后续语义/IR 对接。

当前交付版为了尽快形成完整可评测闭环，将 Lexer、Parser、AST、符号/常量处理和 RISC-V32 代码生成集中在 `src/main.cpp` 中实现。原计划中的独立 IR/Optimizer 模块保留为后续重构方向；当前 `-opt` 已接入常量表达式折叠，正确性优先于激进优化。

### 2.2 公共诊断

所有阶段统一使用 `Diagnostic` 表示错误，记录：

- 错误类型：词法、语法、语义、内部错误。
- 源码位置：行号、列号、字符偏移。
- 错误消息。

评测用例保证合法，因此错误恢复不是核心目标。实现中遇到错误可以向 `stderr` 输出信息并返回非零状态；合法输入必须稳定生成汇编。

## 3. 词法分析

### 3.1 Token 类型

Token 覆盖：

- 关键字：`const`、`int`、`void`、`if`、`else`、`while`、`break`、`continue`、`return`。
- 标识符：`ID`。
- 整数：`NUMBER`。
- 运算符：`+`、`-`、`*`、`/`、`%`、`!`、`&&`、`||`、`<`、`>`、`<=`、`>=`、`==`、`!=`、`=`。
- 分隔符：`;`、`,`、`(`、`)`、`{`、`}`。
- 结束符：`EndOfFile`。

### 3.2 处理规则

- 忽略空白字符。
- 支持 `//` 单行注释和 `/* ... */` 多行注释。
- `NUMBER` 按任务定义支持可选负号，但实现时要谨慎处理 `a-1` 和 `-1`：
  - 词法层优先把 `-` 作为运算符。
  - Parser 在一元表达式中处理负号。
  - 如果输入中出现 `int x = -1;`，解析为一元负号加数字，语义等价。

## 4. 语法分析与 AST

### 4.1 Parser 策略

采用递归下降，并为表达式拆分不同优先级函数：

```text
parseCompUnit
parseDecl
parseConstDecl
parseVarDecl
parseFuncDef
parseBlock
parseStmt
parseExpr
parseLOrExpr
parseLAndExpr
parseRelExpr
parseAddExpr
parseMulExpr
parseUnaryExpr
parsePrimaryExpr
```

左递归文法在实现中改写为循环：

```text
AddExpr -> MulExpr (('+' | '-') MulExpr)*
MulExpr -> UnaryExpr (('*' | '/' | '%') UnaryExpr)*
```

### 4.2 顶层声明与函数区分

看到 `int` 或 `void` 后，根据后续 Token 判断：

- `int ID (` 或 `void ID (`：函数定义。
- `const int ID = ... ;`：常量声明。
- `int ID = ... ;`：变量声明。

ToyC 要求所有变量和常量声明必须初始化，因此 AST 中声明节点始终持有初始化表达式。

### 4.3 AST 节点

核心节点：

- `CompUnit`：顶层声明和函数列表。
- `Decl`：变量声明或常量声明，带作用域上下文。
- `FuncDef`：返回类型、函数名、参数列表、函数体。
- `Block`：语句列表。
- `Stmt`：块、空语句、表达式语句、赋值、声明、条件、循环、跳转、返回。
- `Expr`：字面量、变量引用、一元、二元、函数调用。

AST 只表达语法结构，不直接存储后端寄存器或栈偏移。语义阶段可给节点补充符号引用和常量值。

## 5. 语义分析

### 5.1 符号系统

符号类型：

- `VariableSymbol`：变量，记录是否全局、是否常量、栈偏移或全局标签。
- `FunctionSymbol`：函数，记录返回类型、参数类型和参数名。
- `ConstValue`：编译期常量值。

作用域：

- 全局作用域保存全局变量、全局常量和函数。
- 函数作用域保存形参。
- 每个 Block 建立一个嵌套作用域。
- 内层作用域允许屏蔽外层变量、常量或形参。
- 同一作用域内不允许重名。

### 5.2 检查规则

必须检查：

- 程序存在 `int main()`，且无参数。
- 函数名唯一，函数只能在全局作用域定义。
- 函数调用必须发生在被调函数声明之后；函数自身内部允许递归调用。
- 变量、常量、函数使用必须在声明之后。
- 常量初始化表达式必须可在编译期求值。
- 常量不能作为赋值左值。
- `break` 和 `continue` 只能出现在循环内部。
- `return` 与函数返回类型匹配。
- `int` 函数所有可能路径必须返回 int。
- `void` 函数可以无返回；有返回时不能带表达式。
- void 函数调用不能作为条件、赋值右值或表达式值使用。

### 5.3 常量求值

实现 `ConstEvaluator`，支持：

- 数字字面量。
- 已声明常量引用。
- 一元 `+`、`-`、`!`。
- 二元算术、关系、逻辑运算。
- 短路逻辑在常量求值时同样遵守。

全局常量和局部常量都可进入常量表，供后续表达式折叠和代码生成使用。

## 6. 后续中间表示 IR 设计

### 6.1 选择

当前交付版采用 AST 直接生成 RISC-V32 的方式，以保证功能闭环。后续若继续优化和拆分模块，建议引入简单三地址码 IR，降低 AST 到 RISC-V32 的耦合。IR 不追求复杂 SSA，重点是清晰表达：

- 算术和关系运算。
- 条件跳转和无条件跳转。
- 函数调用和返回。
- 局部变量读写。
- 全局变量读写。
- 标签。

### 6.2 IR 指令草案

```text
Function name(params)
EndFunction

Label L
Jump L
BranchZero value, L
BranchNonZero value, L

Assign dst, src
Binary dst, op, lhs, rhs
Unary dst, op, value

LoadLocal dst, symbol
StoreLocal symbol, value
LoadGlobal dst, symbol
StoreGlobal symbol, value

Call dst?, function, args
Return value?
```

IR 值类型：

- 常量立即数。
- 临时变量。
- 局部符号。
- 全局符号。

### 6.3 短路逻辑

逻辑与、逻辑或不直接降为普通二元运算，而是由 IRBuilder 生成控制流：

- `a && b`：若 `a` 为 0，结果直接为 0；否则计算 `b`。
- `a || b`：若 `a` 非 0，结果直接为 1；否则计算 `b`。

## 7. RISC-V32 代码生成

### 7.1 汇编结构

输出包含：

```asm
    .data
global_name:
    .word value

    .text
    .globl main
function_name:
    ...
```

全局变量放入 `.data`，全局常量可直接作为立即数使用；如果后续需要地址，也可以放入只读标签。

### 7.2 调用约定

使用常见 RISC-V 调用约定：

- 返回值：`a0`。
- 参数：优先使用 `a0` 到 `a7`，超过 8 个参数时放栈上。
- 临时寄存器：`t0` 到 `t6`。
- 函数入口保存 `ra` 和必要的 `s` 寄存器。
- 栈保持 16 字节对齐。

ToyC 没有数组和指针，局部变量都可分配固定栈槽。第一版可以采用保守策略：临时值也落栈，保证正确性；优化阶段再减少访存。

### 7.3 栈帧

每个函数生成：

```asm
addi sp, sp, -frame_size
sw ra, frame_size-4(sp)
...
lw ra, frame_size-4(sp)
addi sp, sp, frame_size
ret
```

局部变量、形参备份、临时值统一由 `StackFrame` 分配偏移。

### 7.4 控制流

`if/else`、`while`、`break`、`continue` 全部转为标签和跳转。循环生成时维护循环上下文：

- `continueLabel`：跳回条件判断。
- `breakLabel`：跳出循环。

### 7.5 除法和取模

使用 RISC-V `div` 和 `rem` 指令。任务保证测试用例没有未定义行为，因此不需要运行时检查除零。

## 8. 优化方案

优化仅在传入 `-opt` 时启用，默认路径优先保证正确性。

当前交付版已经在 `-opt` 下启用常量表达式折叠。以下优化项是后续提高性能分数的扩展方向。

第一阶段优化：

- 常量折叠：表达式和 IR 层都可处理。
- 常量传播：局部基础块内传播不会被修改的常量。
- 死代码删除：删除无副作用且结果未使用的临时计算。
- 跳转简化：删除跳到下一条标签的无条件跳转。

第二阶段优化：

- 局部公共子表达式消除。
- 简单寄存器复用，减少临时值落栈。
- 基础块内 peephole：如 `addi x, x, 0`、连续 move 等。

优化正确性原则：

- 函数调用默认视为有副作用，不删除。
- 全局变量读写默认保守处理。
- 分支和循环边界处不做跨基本块激进传播。

## 9. 实现顺序

### 第 1 步：最小正确闭环

目标：

- `int main(){ return 0; }`
- `int main(){ return 1 + 2 * 3; }`

任务：

- Lexer 基础 Token。
- Parser 支持函数、Block、return、表达式。
- AST 与常量表达式求值。
- 后端输出 main 返回值。

### 第 2 步：局部变量与赋值

目标：

- 支持局部变量声明、初始化、赋值、读取。
- 支持算术、关系、逻辑表达式。

任务：

- 符号表与作用域。
- 栈槽分配。
- 表达式 IR 和代码生成。

### 第 3 步：语句与控制流

目标：

- 支持 `if/else`、`while`、`break`、`continue`、空语句、表达式语句。

任务：

- 控制流 IR。
- 循环上下文语义检查。
- 标签生成和跳转代码生成。

### 第 4 步：函数

目标：

- 支持普通函数、参数、返回值、void 函数、递归。

任务：

- 函数符号表。
- 参数传递和栈帧。
- 调用表达式和返回路径检查。

### 第 5 步：全局变量和常量

目标：

- 支持全局变量、全局常量、局部常量。

任务：

- 全局初始化常量求值。
- `.data` 输出。
- 全局变量读写。
- 常量折叠接入。

### 第 6 步：优化与报告

目标：

- 接入 `-opt`。
- 提供性能对比样例和实践报告素材。

任务：

- IR 优化 Pass。
- 后端 peephole。
- 测试记录和报告章节整理。

## 10. 测试策略

每类功能至少提供正向样例：

- `smoke`：最小 main、常量返回。
- `expr`：优先级、一元、关系、逻辑短路。
- `decl`：局部变量、常量、嵌套作用域、屏蔽。
- `stmt`：if/else、while、break、continue、return。
- `func`：参数、返回值、void、递归、调用顺序。
- `global`：全局变量、全局常量、读写混合。
- `opt`：常量折叠、死代码、循环性能样例。

本地测试建议流程：

1. 编译 ToyC 到汇编。
2. 用 RISC-V 工具链汇编和链接。
3. 运行可执行文件，检查退出码。
4. 与等价 C 程序的退出码对比。

若本地缺少 RISC-V 工具链，至少先做前端/IR 字符串快照测试，并在具备环境后补汇编运行测试。

## 11. 关键风险

| 风险 | 应对 |
| --- | --- |
| Parser 和 AST 频繁变化 | 先固定节点类型和表达式优先级，再接语义 |
| 栈帧和调用约定出错 | 先支持少量参数，再扩展到超过 8 个参数 |
| 短路逻辑结果错误 | 短路表达式单独生成控制流，不当作普通二元运算 |
| 常量表达式边界复杂 | 单独实现 `ConstEvaluator`，不要混在代码生成中 |
| 优化破坏正确性 | 优化默认关闭，`-opt` 分 Pass 开启并配回归测试 |
| 报告缺少数据 | 每完成一个阶段同步记录设计、样例和问题修复 |
